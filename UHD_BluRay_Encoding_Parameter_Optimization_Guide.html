UHD 蓝光编码参数优化指南

副标题：在 Scenarist UHD MUX 不出现 buffer_underflows 的前提下，最大化利用不同 UHD 碟片的 M2TS 传输速率

文件名：UHD_BluRay_Encoding_Parameter_Optimization_Guide.html ｜ 生成日期：2026-06-27

0. 核心结论

你的方法可以概括为：先用理论约束确定安全外包络，再用多轮 Scenarist UHD MUX 实验把外包络收敛为生产参数。关键不是把编码器瞬时峰值直接推到碟片 M2TS 极限，而是把 M2TS_maxRate 拆成“音频最大占用 + 视频安全预算 + 极小校验余量”，再用固定的 VBV 安全模型把视频预算转换为 BL/EL 的 max_vbv_data_rate、vbv_buffer_size 和平均 data_rate。

最终形成了两个互补配置：MEL 单碟发行版利用 BD-ROM 100GB 的容量，以 109 Mbps M2TS 上限为外约束，将视频安全预算锁定为 96 Mbps；FEL 三碟测试版利用 two-zoned 66GB 的 127.9 Mbps HTR 传输能力，为大量音频轨和 FEL 残差层腾出传输预算，同时仍把视频安全预算固定在 96 Mbps，避免 MUX 端发生读缓冲欠载。

1. 理论约束：先定义可用传输外包络

1.1 BDAV MPEG-2 Transport Stream 是 MUX 约束的实际对象

UHD Blu-ray 的片流不是只看 HEVC elementary stream 的峰值，而是把视频、音频、图形字幕、菜单图形等 elementary streams 封装到 BDAV MPEG-2 Transport Stream 中。BDA 白皮书描述了 BDAV MPEG-2 TS 的 192-byte source packet、6144-byte aligned unit 结构，并给出不同 disc type 与 zone 对应的最大复用速率。这个速率才是 Scenarist UHD MUX 判断读缓冲安全性的外侧边界。

碟片类型可用容量与区域采用的 M2TS 上限工程用途
BD-ROM 100GB (122.546 Mbps)95,466.00 MB109 MbpsMEL 单碟发行版。容量是主要优势。
BD-ROM 66GB (two-zoned disc)58,552.48 MB127.9 MbpsFEL 三碟测试版。HTR 传输速率是主要优势。
BD-ROM 100GB (two-zoned disc)87,828.72 MB127.9 Mbps理论上可同时获得大容量与高 HTR 速率，但会受 HTR 文件分配规则和容量折损约束。

1.2 HEVC 主视频仍受 100 Mbps 级别约束

BDA 白皮书的 Primary Video 表给出 UHD BD HEVC Main 10、High Tier、Level 5.1 的 3840×2160、59.94p 支持，并列出 HEVC primary video 的最大码率为 100 Mbps。因此，BL、MEL、FEL 的 maxRate 设计不能只满足 M2TS 外包络，还必须满足单层或分层后的 elementary stream 上限。

1.3 Dolby Vision Profile 7 的 BL/EL 分配原则

Dolby Vision UHD Blu-ray Profile 7 是双层编码：BL 是 HDR10 兼容的 10-bit full-picture layer，EL 携带残差和动态元数据。MEL 的残差信号为零或最小化，FEL 的残差信号非零，可用于更完整地重建 12-bit 高峰值母版信息。Dolby 工作流指南给出的经验区间是：MEL 使用 98/2 到 99.5/0.5 的 BL/EL 比例；FEL 使用 80/20 到 95/5 的 BL/EL 比例。你的计算器把这些比例转化为可验证的整数 Kbps 参数。

注：Dolby 指南同时指出，对于 50 fps 或更高 UHD 内容，在某些映射工作流下推荐或要求使用 MEL 以提升处理效率。因此，本文把 59.94p FEL 方案定位为“工程测试版/压力测试版”，而 MEL 方案定位为更稳妥的单碟发行配置。

2. VBV 安全模型：把 M2TS 预算转化为编码器参数

你的 RateRules 把 MUX 可用视频预算抽象为 VBV_safeRate。它不是编码器的瞬时峰值，而是对 M2TS 读缓冲更友好的安全预算。核心模型如下：

VBV_maxRate = 0.9 × VBV_safeRate

VBV_bufferSize = 1/3 × VBV_maxRate

VBV_peakRate = VBV_maxRate + VBV_bufferSize

VBV_peakRate + 2 × VBV_maxRate = 3 × VBV_safeRate

当 VBV_safeRate = 96 Mbps 时：

总视频参数数值作用
VBV_safeRate96,000 Kbps与音频最大占用相加后用于 M2TS 安全校验。
VBV_maxRate86,400 Kbps编码器 VBV 最大数据率。
VBV_bufferSize28,800 Kbps缓冲裕量；用于吸收瞬态复杂度。
VBV_peakRate115,200 Kbps理论瞬时峰值，不直接等同于 M2TS 预算。

这个模型的工程意义是：编码器可以有瞬态复杂度吸收能力，但 MUX 的连续读出压力仍按 96 Mbps 安全视频预算设计。这正是避免 Scenarist UHD MUX buffer_underflows 的关键。

3. 工程化优化流程：理论研究 + 多次实验 → 生产实践

  1. 确定碟片外包络。先选 disc type 与 zone，得到容量上限和 M2TS_maxRate。普通 100GB 取 109 Mbps；two-zoned 66GB 与 100GB 的 HTR 区可取 127.9 Mbps。
  2. 音频先行。把所有随主片复用的音频轨按最大码率求和。TrueHD、DTS-HD MA 这类 VBR 音轨使用实测最大值，不用平均值。PGS 字幕和图形流若码率显著，也应纳入保守预算。
  3. 反推出视频安全预算。Video_safeRate ≤ M2TS_maxRate − Audio_maxRate − Guard。你的两个方案都把 Video_safeRate 收敛到 96 Mbps。
  4. 套用 VBV 模型。由 96 Mbps 得到总 max_vbv_data_rate = 86.4 Mbps、总 vbv_buffer_size = 28.8 Mbps。
  5. 按 Dolby Vision 层类型分配 BL/EL。MEL 使用 99.5/0.5，使绝大部分码率进入 BL；FEL 使用 95/5，使 EL 保留必要残差预算，同时避免 BL 超过 100 Mbps 上限。
  6. 用容量决定平均 data_rate。MEL 单碟以容量压满为目标，平均视频为 72 Mbps；FEL 三碟测试版以传输压力与 FEL 实验为目标，平均视频可设到 86.4 Mbps。
  7. 反复 MUX 验证。每轮都用实际编码流进入 Scenarist UHD MUX，检查 buffer_underflows、M2TS 校验码率、Clip/Playlist 播放和实机兼容；只把通过 MUX 的参数固化为生产规则。

4. 音频流信息与复用预算

音频预算按“参与主片 M2TS 复用的音频流最大码率”计算。VBR 轨道使用编码后的最大值，CBR 轨道使用恒定码率；这样得到的总和用于与视频 VBV_safeRate 相加，形成 Scenarist UHD MUX 的校验压力。

5. 案例 A：MEL 单碟发行版

目标

在 BD-ROM 100GB、109 Mbps M2TS 上限内，尽量压满单碟容量，同时保持 Scenarist UHD MUX 不报 buffer_underflows。

工程判断

MEL 的 EL 负担极小，适合把码率集中到 BL；音频轨数量相对克制，因此可用 100GB 单碟容量完成发行版。

项目数值解释
碟片BD-ROM 100GB (122.546 Mbps) ×1有效容量 95,466.00 MB；M2TS 上限 109 Mbps。
音频流最大比特率12.969 MbpsTrueHD Atmos、DD+ Atmos、DTS-HD MA 2.0 等全部复用音频的最大占用。
视频安全比特率96.000 Mbps由 109 − 12.969 ≈ 96.031 Mbps 反推出，取整数 96 Mbps。
M2TS 校验比特率108.969 Mbps音频最大 + 视频安全 = 108.969 Mbps。
对 109 Mbps 的余量31 Kbps极窄余量，必须依赖保守 VBV 和实际 MUX 验证。
容量利用94,673.04 / 95,466.00 MB = 99.17%剩余 792.96 MB，接近单碟容量上限。

最终编码参数

data_rate         = 71640 (BL),   360 (MEL)  # total 72,000 Kbps
max_vbv_data_rate = 85968 (BL),   432 (MEL)  # total 86,400 Kbps
vbv_buffer_size   = 28656 (BL),   144 (MEL)  # total 28,800 Kbps
BL/MEL ratio      = 99.5% / 0.5%

该配置的核心是容量优先、M2TS 临界控制。平均视频 72 Mbps 使单碟容量利用率达到 99.17%；而总 max_vbv_data_rate 仍限制在 86.4 Mbps，避免把短时复杂度直接推成 MUX 读出压力。MEL 只给 0.5% 的码率，是因为 MEL 不承担有效残差重建任务，主要服务 Dolby Vision 元数据与最低增强层结构。

6. 案例 B：FEL 三碟测试版

目标

验证 FEL + 大量音频轨 + 59.94p UHD 的高压力组合，在 two-zoned 66GB HTR 速率下是否可稳定 MUX。

工程判断

FEL 与多音轨显著抬高传输预算，普通 109 Mbps 外包络不足；使用 127.9 Mbps HTR zone，并通过三碟拆分解除容量压力。

项目数值解释
碟片BD-ROM 66GB two-zoned disc ×3单碟有效容量 58,552.48 MB；HTR M2TS 上限 127.9 Mbps。
音频流最大比特率31.232 Mbps包含 TrueHD Atmos、DD+ Atmos、DD+ 7.1 EX、DTS-HD MA 7.1/5.1/2.0 等多音轨。
视频安全比特率96.000 Mbps由 127.9 − 31.232 = 96.668 Mbps 反推出，保守取 96 Mbps。
M2TS 校验比特率127.232 Mbps音频最大 + 视频安全 = 127.232 Mbps。
对 127.9 Mbps 的余量668 Kbps比 MEL 单碟宽，但仍需要 Scenarist UHD MUX 实测确认。
容量利用47,633.54 / 58,552.48 MB = 81.35%剩余 10,918.94 MB；三碟方案主要服务传输压力与实验弹性。

最终编码参数

data_rate         = 82080 (BL),  4320 (FEL)  # total 86,400 Kbps
max_vbv_data_rate = 82080 (BL),  4320 (FEL)  # total 86,400 Kbps
vbv_buffer_size   = 27360 (BL),  1440 (FEL)  # total 28,800 Kbps
BL/FEL ratio      = 95% / 5%

该配置的核心是传输速率优先、容量由分碟解决。FEL 残差层比 MEL 更有意义，不能把 EL 压到 0.5%；因此取 95/5，把 FEL 层维持在 4.32 Mbps，同时 BL 维持 82.08 Mbps，二者均低于 100 Mbps 约束。由于音频最大占用高达 31.232 Mbps，只有 two-zoned disc 的 127.9 Mbps HTR 外包络能给 96 Mbps 视频安全预算留下足够空间。

7. 可复用的生产实践规则

规则执行方式目的
先算 M2TS，再算视频Video_safeRate = M2TS_maxRate − Audio_maxRate − Guard避免把视频参数孤立优化后在 MUX 阶段失败。
音频用最大值，不用平均值TrueHD Atmos、DTS-HD MA 取编码后的最大码率；Dolby Digital、Dolby Digital Plus CBR 取恒定码率。Scenarist 的读缓冲风险来自峰值复用压力。
固定 96 Mbps 视频安全档在 109 Mbps 和 127.9 Mbps 两类外包络下统一使用 96 Mbps。形成跨方案可复用的 VBV 参数档。
总 max_vbv 固定 86.4 Mbps按 0.9 × 96 Mbps 设置。给 MUX 和解码器 HRD 留出调度空间。
MEL 以 BL 为主优先 99.5/0.5；必要时回退到 99/1、98.5/1.5 或 98/2。把码率用于可直接改善 HDR10 BL 的画质。
FEL 保留残差预算在 80/20 到 95/5 内选择满足 BL ≤ 100 Mbps 的最高 BL 比例。兼顾 FEL 残差信息和 BL 上限。
容量和速率分开优化100GB MEL 单碟解决容量；66GB two-zoned FEL 多碟解决传输峰值。避免把“容量够”和“M2TS 不欠载”混为一谈。
最终以 MUX 实验为准音频、字幕、章节拆分、码率或 BL/EL 比例变更后，均重新进入 Scenarist UHD MUX 校验。把理论模型变成生产可交付参数。

建议的参数档模板

# Universal 96 Mbps video-safe profile
VBV_safeRate      = 96,000 Kbps
VBV_maxRate       = 86,400 Kbps
VBV_bufferSize    = 28,800 Kbps
VBV_peakRate      = 115,200 Kbps

# MEL production profile
BL_MEL_ratio      = 99.5 / 0.5
BL max/buffer     = 85,968 / 28,656 Kbps
MEL max/buffer    =    432 /    144 Kbps

# FEL stress-test profile
BL_FEL_ratio      = 95 / 5
BL max/buffer     = 82,080 / 27,360 Kbps
FEL max/buffer    =  4,320 /  1,440 Kbps

8. 验证与 QC 清单

8.1 编码前

  • 核对母版元数据：P3-D65 within BT.2020 container、PQ ST 2084、ST 2086、MaxCLL、MaxFALL、mastering display luminance。
  • 核对 Dolby Vision RPU：L0 mastering monitor、L1 shot metadata、L5 active area、L6 MaxCLL 与 MaxFALL。
  • 确认 59.94p UHD 内容在目标工具链中对 MEL/FEL 的支持边界；FEL 59.94p 应按测试版处理。

8.2 编码后

  • 用 Dolby Vision Bitstream Verifier 与 Blu-ray Verifier 检查 BL、EL、RPU 的语法和元数据一致性。
  • 核对 BL/EL 帧同步。EL 不同步通常表现为重影；元数据不同步通常表现为镜头中间的异常亮度跳变。
  • 抽查 HDR10 BL 画质，避免为 Dolby Vision 层牺牲非 Dolby Vision 播放兼容体验。

8.3 Scenarist UHD MUX

  • 使用实际 encoded elementary streams，而不是估算码率文件。
  • 检查 buffer_underflows、M2TS check-rate、clip duration、playlist、章节点、seamless connection。
  • 若出现 underflow，优先按顺序处理：降低 Video_safeRate → 降低平均 data_rate → 降低音频峰值或减少音轨 → 调整分碟或改用 two-zoned disc。

8.4 实机播放

  • 分别测试 Dolby Vision 输出、HDR10 fallback、无 Dolby Vision 设备播放。
  • 测试 PGS 繁简字幕在 600 cd/m² CLUT 下的亮度、边缘和闪烁。
  • 如果有大面积菜单图形，按 Dolby 建议验证 Graphics Priority mode；仅字幕一般不应触发 graphics priority。
